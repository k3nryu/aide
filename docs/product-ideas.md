# 产品想法记录

这份文档先记录当前产品方向想法，不急着变成完整需求。后续可以再拆成数据模型、API、UI、路线图。

## Task

Task 不应该只是简单的 todo list，而是一个个人执行系统。它需要支持定期任务、不定期任务、计划与复盘、自动安排，以及和 not-to-do 的冲突提醒。

### 任务类型

- 定期任务
- 不定期任务

定期任务示例：

- 每天学习 X 个单词。
- 每周四、周六上游泳课。
- 每半年更新一次定期券。
- 每个月检查一次工资卡。
- 每年农历某月某日的生日，比如自己的生日、老婆的生日，并且可以提前 X 个月开始提醒和准备。

每日简报应该能把当天需要关注的任务列出来，包括由定期任务自动生成或推导出来的今日事项。

### PDCA

任务需要支持 PDCA：

- Plan：计划要做什么。
- Do：执行任务。
- Check：检查结果和执行情况。
- Act：调整任务、习惯或后续计划。

这可以让 Aide 不只是记录任务，还能帮助用户持续改进执行方式。

### 四象限分类

任务需要支持重要/紧急四象限：

- 重要且紧急
- 重要但不紧急
- 不重要但紧急
- 不重要且不紧急

Aide 未来应该根据任务的重要程度、紧急程度、截止日期、定期规则和用户状态，自动帮助安排任务。

### 大任务和 WBS

大的任务应该可以做 WBS 拆分。

未来 UI 可以把大任务可视化为树状结构、清单、时间线或其他适合执行的结构，而不是只显示成一个模糊的大 todo。

### 和 Not-To-Do 的冲突提醒

如果 task 违背了 not-to-do，Aide 应该给出警告。

例如，一个任务如果违反法律边界、道德良心、公司规则、家庭原则或身体健康原则，系统应该在计划或执行前提醒用户。

## 行为记录

除了 task，Aide 还应该支持记录用户实际做过的事情。

这些事情不一定提前出现在 task list 里，也不一定需要计划，但它们对长期分析很有价值。

示例：

- X 点上了一次厕所。
- X 日去了一趟 X 地。
- X 点吃了某种食物。
- X 点睡觉或起床。
- X 日运动了一次。
- X 日出现了某种身体不舒服。
- X 日做了某个临时决定或临时行动。

这些记录更接近 life log 或 activity log，不应该强行当成 todo。

未来用途：

- 推测健康程度。
- 分析行为趋势。
- 发现不良习惯。
- 发现和强化良好习惯。
- 辅助生成每日、每周、每月复盘。
- 和 task、not-to-do、财务记录、AI memory 产生关联。

行为记录需要尽量低摩擦，最好可以快速输入一条自然语言记录，之后再由系统自动分类、打标签或生成结构化数据。

## Not-To-Do

Not-to-do 应该是一等公民，不只是普通备注。它代表用户明确不想做、不能做、或不应该做的事情。

初始分类：

- 违法的事情不做。
- 违背道德良心的事情不做。
- 违背公司规则的事情不做。
- 对家人不好的事情不做。
- 对自己身体不好的事情不做。

用户应该可以随时在这些分类下添加新的 not-to-do。

这些规则未来应该可以用于检查 task，并在发现冲突时给出提醒。

## 财务

财务功能的长期方向可以接近 Money Forward。

想法包括：

- 通过各个银行的 API 获取或同步数据。
- 可视化当前财务状态。
- 记录收入、支出、账户、资产等信息。
- 记录财富积累目标。
- 计算距离实现财富自由还差多少钱。

这个方向会涉及隐私、安全、银行 API、数据导入和可视化，属于比较大的功能模块。

## Thoughts

用户已经有 Joplin 笔记本，可以随时记录想法。

所以 Aide 里的 thoughts 功能目前可以不是核心优先级。未来可以考虑：

- 保留一个轻量 capture 功能。
- 以后和 Joplin 集成。
- 从 Joplin 中导入部分笔记进入 Aide memory。
- 只把需要 follow-up、转成任务、或进入 AI memory 的内容放进 Aide。

## 当前优先级倾向

根据这些想法，当前产品优先级可以倾向于：

1. 强化 task 系统：定期任务、每日简报、PDCA、四象限、WBS。
2. 增加行为记录：记录实际发生过的事情，用于健康、习惯和趋势分析。
3. 强化 not-to-do：分类规则、随时添加、task 冲突提醒。
4. 扩展财务：财务状态可视化、财富目标、财富自由差距计算。
5. 降低 thoughts 优先级：先作为可选功能，未来考虑 Joplin 集成或 AI memory 输入源。
